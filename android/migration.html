Migration Guide

Migrating from a previous Speedify SDK

Prior to Speedify SDK v12.5.1, the Safebrowsing functions were parts of the Speedify SDK. As of v12.5.1, the Safebrowsing functionality has been split out into the Cujo Safebrowsing SDK.

First, the new Safebrowsing SDK needs to be added as a dependency to the app and the Speedify SDK dependency must be using v12.5.1 or higher.

Safebrowsing functions that were previously called on the SpeedifySDK instance are now available on the Safebrowsing instance. Some functions have been renamed to remove unnecessary mentions of "safebrowsing" in the name.

See the SDK initialization page for details on getting a Safebrowsing SDK instance and setting handlers. To get Safebrowsing callbacks, such as OnSafeBrowsingConnectError in the app, a SafeBrowsingHandler can be registered with the Safebrowsing instance.

Safebrowsing.setHandler(handler);

The following functions from the SpeedifySDK were renamed in the Safebrowsing SDK:

  • startSafeBrowsing -> start
  • stopSafeBrowsing -> stop
  • initSafeBrowsing -> initWithConfig
  • setSafeBrowsingConfig -> setConfig
  • setSafeBrowsingEnabled -> setEnabled
  • setSafeBrowsingLogLevel -> setLogLevel
  • setSafeBrowsingSafeNetworkOverride -> setSafeNetworkOverride
  • reportSafeBrowsingHomeNetwork -> reportHomeNetwork

As an example, the following call to start safebrowsing on the SpeedifySDK

SpeedifySDK.getInstance().startSafeBrowsing(config);

migrates to

Safebrowsing.getInstance().start(config);
