Speedify SDK for Desktop  10.8.1
What's New

Speedify SDK for Desktop 10.8

  • The language used for error messages can now be set with SpeedifySDK_SetLanguage. This takes a 2-character ISO language code. Supported languages include 'en', 'ar, 'de', 'es', 'fr', 'pt', 'ro', 'ru', 'tr', 'zh'.
  • Error messages may contain a tag {speedifyProductName}. You should replace this with your product name.

Speedify SDK for Desktop 10.7.1

  • No new features. This is a stability and performance release.

Speedify SDK for Desktop 10.7

  • Header compression now defaults to on.
  • Disconnect Reasons have been expanded to include DR_DNSTIMEOUT and DR_TUNNELTIMEOUT.
  • Native 64-bit support on Windows.
    • Note that everything under the 'tap-driver/' folder is architecture-dependent, and can only be used on the architecture it was designed for. The x86 versions will not function properly if run under x64 or arm64.

Speedify SDK for Desktop 10.6

  • Calling SpeedifySDK_Logout will now reset your connect method to ACM_CLOSEST.
  • Header compression can be enabled via SpeedifySDK_SetHeaderCompression. Currently support extends to IPv4, UDP, and partial TCP (ports only).

Speedify SDK for Desktop 10.5

  • Speedify SDK now supports a centralized OAuth gateway for authentication. Setup the gateway URI with SpeedifySDK_SetDirectoryGatewayURI before logging in with SpeedifySDK_LoginOAuth.
  • Log settings in the daemon service are now configurable through SpeedifySDK_SetDaemonLogSettings.
  • Log files may be deleted with SpeedifySDK_EraseAllLogFiles. Note that the current active log file is not erased.
  • Credentials, like OAuth tokens and passwords, can be restricted from being saved locally using SpeedifySDK_SetDoNotStoreCredentials. Note that you will have to manually call a login method on every new SDK daemon run.
  • API Compatibility Changes
    • SpeedifySDK_LoginOauth has been changed to SpeedifySDK_LoginOAuth.
  • Deprecations
    • SpeedifySDK_DirectoryDomainCallback has been deprecated. Use SpeedifySDK_DirectorySettingsCallback.
    • SpeedifySDK_DirectoryDomainResult has been deprecated. Use SpeedifySDK_DirectorySettingsResult.
    • SpeedifySDK_GetDirectoryDomain has been deprecated. Use SpeedifySDK_GetDirectorySettings.
    • SpeedifySDK_SetCrashReports/crashReports - Crash reporting has been removed from the SDK for several releases.

Speedify SDK for Desktop 10.4

  • Proxy Mode - You may not start a local-proxy-only VPN by calling SpeedifySDK_ConnectProxy. All traffic through the VPN will be sent out via the Local Proxy over the default network interface.
  • You can now disable ESNI in directory and speed server requests with SpeedifySDK_SetESNIEnabled if your custom directory server does not support ESNI.
  • The reason for the last disconnect may be queried via SpeedifySDK_GetLastDisconnectEvent.

Speedify SDK for Desktop 10.3.1

  • No new features. This is a stability and performance release.

Speedify SDK for Desktop 10.3

  • On Windows, Speedify will set DisableDefaultRoute to true on all network interfaces while running so that traffic doesn't leak around the VPN. This can interfere with hosting servers on the speedify device. This functionality can be disabled via SpeedifySDK_SetIpLeak.
  • Fixes an issue with calling other SDK methods from within an asynchronous SDK callback.

Speedify SDK for Desktop 10.2

  • API Compatibility Changes
    • SpeedifySDK_ProxyMatchMode
      • PMM_WHITELIST and PMM_BLACKLIST have been renamed to PMM_ALLOWLIST and PMM_DENYLIST.
    • SpeedifySDK_FirewallMode
      • FWM_WHITELIST and FWM_BLACKLIST have been renamed to FWM_ALLOWLIST and FWM_DENYLIST.

Speedify SDK for Desktop 10.1

  • Support for virtual interfaces on Linux.

Speedify SDK for Desktop 10.0.1

  • No new features. This is a stability and performance release.

Speedify SDK for Desktop 10.0

  • New Session Stats API
    • The named time period members (current, day, week, month, total) are removed in favor of custom time periods. The periods member now contains a list of all requested time periods. Use the period member of SpeedifySDK_SessionStats to identify the period the stats are for.
    • Use SpeedifySDK_EnableStatUpdates to specify wanted time periods.
  • Session Stats now include SpeedifySDK_StreamingSessionStats for each period.
  • When using asynchronous SDK methods, you can now wait for the callback to occur via SpeedifySDK_WaitOnCallback. This will wait in the current thread until the desired callback has returned.
  • API Compatibility Changes
    • SpeedifySDK
      • SpeedifySDK_EnableConnectionStatUpdates is deprecated. Use SpeedifySDK_EnableStatUpdates instead.
    • SpeedifySDK_SessionStatPeriods
      • The named time period members (current, day, week, month, total) are removed in favor of custom time periods. The periods member now contains a list of all requested time periods. Use the period member of SpeedifySDK_SessionStats to identify the period the stats are for.
    • StreamingStats
      • Stats on total streams and events have been moved in to SpeedifySDK_StreamingSessionStats available in SpeedifySDK_SessionStats

Speedify SDK for Desktop 9.9

  • The Local Proxy API now supports matching by TCP and UDP port: SpeedifySDK_SetLocalProxyPorts.

Speedify SDK for Desktop 9.8

Speedify SDK for Desktop 9.7

  • No new features. This is a stability and performance release.

Speedify SDK for Desktop 9.6

  • Speedify has a new Streaming Mode to better support video streamers. This can be set via SpeedifySDK_SetMode with BMODE_STREAMING. Statistics on individual streams are available from SpeedifySDK_GetStreamingStats, and are provided once per second when enabling SpeedifySDK_EnableConnectionStatUpdates.

Speedify SDK for Desktop 9.5

  • Session stats in SpeedifySDK_SessionStatsResult are now tracked over multiple time periods, including current session, last day, last week, last month, and total.
  • API Compatibility Changes
    • SpeedifySDK_SessionStatsResult
      • Now contains a SpeedifySDK_SessionStatPeriods object instead of a SpeedifySDK_SessionStats to accommodate stats over various time periods.

Speedify SDK for Desktop 9.4

  • SpeedifySDK_SetEnableDefaultRoute can be used to disable Speedify from obtaining the default route and handling all Internet traffic.
  • API Compatibility Changes
    • SpeedifySDK_LocalProxyDomainWatchlistItem
      • domains - The domains list has been split in to separate watchedDomains and proxiedDomains lists.

Speedify SDK for Desktop 9.3

  • The Firewall API now supports filtering by IPv6: SpeedifySDK_SetFirewallIPv6. ICMP and other protocols are now also blockable by domain and IP.
  • The Local Proxy API now supports Domain Watchlists. Watchlists are lists of domains grouped by the service they represent, configurable via SpeedifySDK_SetLocalProxyDomainWatchlist. When an attempt to access a watched domain is seen, SpeedifySDK_ListenForLocalProxyDomainWatchlistHit is triggered. SpeedifySDK_SetLocalProxyDomainWatchlistEnable can then be used to enable that service's domains to be sent through the local proxy instead of the VPN.
  • API Compatibility Changes
    • SpeedifySDK_SetFirewallIPv4 and SpeedifySDK_SetLocalProxyIPv4 have been changed to take an array of struct in_addr instead of uint32_t, for consistency with new SpeedifySDK_SetFirewallIPv6 method arguments.

Speedify SDK for Desktop 9.2

  • The Firewall API is used to block traffic over the VPN tunnel.
    • SpeedifySDK_SetFirewallMode
    • SpeedifySDK_SetFirewallDomains
    • SpeedifySDK_SetFirewallIPv4
    • SpeedifySDK_SetFirewallPorts
  • Speedify SDK can now be configured to send certain domains and IP addresses around the VPN and straight out the local network.
    • SpeedifySDK_SetLocalProxyEnabled
    • SpeedifySDK_SetLocalProxyMatchMode
    • SpeedifySDK_SetLocalProxyDomains
    • SpeedifySDK_SetLocalProxyIPv4
  • Speedify SDK now provides SpeedifySDK_SetLocalVendorSettings for saving a custom JSON object for later consumption.
  • API Compatibility Changes
    • IPAddressString has been changed to SpeedifySDK_IPAddressString to match namespace convention with rest of SDK.
